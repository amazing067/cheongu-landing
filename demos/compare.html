CDN 방식 vs @apply 방식 비교

같은 UI를 두 방법으로 구현. 왼쪽은 유틸 클래스를 **직접** 나열(CDN), 오른쪽은 @apply로 만든 **재사용 클래스** 사용.

A) CDN (유틸 직접 나열)

삼성화재
전산 접속 필요서류 안내 보험금 청구서 PDF 다운로드 고객센터 모바일 청구 
<div class="grid items-center gap-4 px-5 py-3 ..."> ... (클래스 매우 길음)

B) @apply (컴포넌트 클래스)

삼성화재
전산 접속 필요서류 안내 보험금 청구서 PDF 다운로드 고객센터 모바일 청구 
.btn { @apply ... }  .row { @apply ... } → HTML은 class="btn btn-primary"처럼 짧음
정리: 두 카드의 스타일은 거의 동일하지만, 우측은 @apply로 공통 클래스를 만들어 HTML이 훨씬 짧고 재사용이 쉽다.
